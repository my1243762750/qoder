## 前端学习与实战项目总计划

> 目标：配合「迷你 Jira」后端，学习 Vue 3 企业级开发。通过 5 个阶段的演进，掌握从环境搭建到复杂业务交互、性能优化及工程化部署的完整前端开发链。

---

## 1. 总体说明

### 1.1 项目整体目标
- **实战目标**：构建一个功能完整、交互流畅的「迷你 Jira」前端系统。
- **学习目标**：
  - 掌握 Vue 3 (Composition API) + Vite 开发基础。
  - 熟练使用 Element Plus 组件库进行 UI 快速搭建。
  - 掌握 Axios 请求封装、拦截器及 JWT 认证流。
  - 理解响应式设计、状态管理（Pinia）及前端路由（Vue Router）。
  - 掌握前端工程化：自动化构建、Docker 部署、性能分析。

---

## 2. 技术栈选型

- **框架**：Vue 3 (SFC + Composition API)
- **构建工具**：Vite
- **状态管理**：Pinia
- **路由管理**：Vue Router
- **UI 组件库**：Element Plus
- **网络请求**：Axios
- **图标**：Element Plus Icons / Iconify

---

## 3. 阶段规划总览

### 3.1 阶段 0：项目初始化 & 基础认知（1～2 天）
- **目标**：跑通 Vite + Vue 3 环境，打通与后端的第一个接口。
- **任务**：
  - 初始化 Vite + Vue 3 项目。
  - 引入 Element Plus 并配置全局/按需加载。
  - 封装 Axios 基础实例（BaseURL, Timeout）。
  - 实现一个简单的 Demo 页面，显示后端 `/api/demo/hello` 的内容。

### 3.2 阶段 1：MVP 版本——核心认证与基础数据（1～2 周）
- **目标**：完成登录注册流及基础的项目/任务展示。
- **任务**：
  - **认证模块**：登录/注册页面（表单校验、Token 存储）。
  - **布局系统**：基础侧边栏 + 顶部导航（Layout 架构）。
  - **项目模块**：项目列表展示、创建项目对话框。
  - **状态同步**：请求头自动携带 JWT Token（Axios 拦截器）。

### 3.3 阶段 2：业务加强 & 交互优化（1～2 周）
- **目标**：实现复杂的数据交互，如分页、筛选及成员管理。
- **任务**：
  - **任务模块**：项目详情页，包含任务列表的分页、状态/优先级筛选、搜索。
  - **操作流转**：点击修改任务状态、指派负责人。
  - **成员管理**：项目内邀请/移除成员的交互。
  - **权限控制**：根据后端返回的角色，控制 UI 元素的显示/隐藏。

### 3.4 阶段 3：工程化 & 功能补全（2 周）
- **目标**：支持文件上传、全局通知，并完成 Docker 化。
- **任务**：
  - **附件系统**：任务详情页支持多文件拖拽上传、下载及预览。
  - **通知系统**：全局 Message 提醒，集成后端异步任务反馈。
  - **持久化优化**：使用 Pinia 管理用户信息和项目状态。
  - **部署**：编写 Nginx 配置与 Dockerfile，实现容器化部署。

### 3.5 阶段 4：性能优化 & 高级特性（2+ 周）
- **目标**：提升加载速度与用户体验，完善测试。
- **任务**：
  - **加载优化**：路由懒加载、按需引入、图片压缩。
  - **交互体验**：骨架屏（Skeleton）、防抖/节流、空状态处理。
  - **质量保障**：编写简单的 Vitest 单元测试。
  - **版本演进**：对接后端 v2 版本接口，处理数据结构变化。

---

## 4. 与后端协同指南

1. **API 对齐**：前端接口路径必须以 `/api` 开头，并严格遵守后端定义的 DTO 结构。
2. **错误处理**：统一处理后端返回的 `code != 0` 的情况，通过 `ElMessage` 弹出后端返回的 `message`。
3. **本地开发**：通过 Vite Proxy 配置解决跨域问题。
