# 迷你 Jira 通用技能（SKILL）

> 说明：本技能文件给所有在本仓库中工作的 AI 使用，帮助它们在实现「迷你 Jira」项目时有固定的工作流程。你可以在对话中用自然语言触发这些“技能模版”，不需要真的输入 /command。

---

## Skill A：实现后端功能（后端实现器）

**触发方式（你可以这样说）：**

- 「按照学习计划文档的阶段 X 第 Y 条，帮我实现 XXX 的后端接口，遵守项目规则。」
- 「帮我实现用户注册 + 登录的后端代码。」
- 「帮我实现项目列表和创建项目接口。」

**AI 执行步骤（必须遵守）：**

1. 打开并阅读本仓库中的整体计划文档（如 `backend-learning-plan.md`）以及 `.qoder/rules/common.md` 中的规则说明，确认：
   - 本次要实现的功能需求。
   - 必须遵守的技术栈和规则。
2. 根据当前项目实际结构选择合适的包路径：
   - Controller 放在业务模块的 `controller` 包下。
   - Service 放在对应 `service` 包中。
   - Repository 放在 `repository` 包中。
   - DTO、Entity 分别放在 `dto`、`entity` 或类似命名包中。
3. 按 Rules 文档要求实现：
   - 使用 DTO 承载请求/响应，不暴露实体。
   - Controller 只负责组装、转发请求，不写复杂业务逻辑。
   - Service 中完成业务规则与权限判断。
   - Repository 中使用 Spring Data JPA 访问数据库。
   - 对外统一返回 `{ code, message, data }` 结构。
   - 对请求参数使用 Bean Validation 做校验。
4. 如涉及数据库结构变化，应在说明中明确：
   - 新增/修改了哪些字段或表。
   - 是否需要你执行建表或迁移操作（并给出 SQL 或迁移脚本草案）。
5. 在回复中只做**高层说明**，不在总结里贴出整段代码：
   - 列出本次创建/修改的类与主要方法名称。
   - 提示你如何在本地运行和测试该功能（接口路径、请求示例等）。

## Skill B：实现或修改前端功能（前端实现器）

**触发方式：**

- 「后端接口已经就绪，帮我在 Vue 前端实现登录页面并接好接口。」
- 「帮我在项目详情页增加任务列表展示和创建任务的表单。」

**AI 执行步骤：**

1. 确认后端接口：路径、HTTP 方法、请求体和响应结构。
2. 在 Vue 3 + Vite 项目中：
   - 使用 Axios 或统一封装的 HTTP 模块调用接口。
   - 按既有项目结构将代码写入对应视图组件 / 路由 / store 中。
   - 使用 Element Plus 组件实现表单、表格、弹窗等 UI。
3. 处理好：
   - 登录成功后 Token 的保存与自动注入（请求拦截器）。
   - 错误状态的展示（如全局消息、表单错误提示）。
4. 在回复说明中：
   - 指明修改了哪些组件/文件。
   - 提供简单的操作步骤（如何在浏览器中验证该功能）。

## Skill C：概念讲解 + 面试强化（讲解器）

**触发方式：**

- 「结合我们这个迷你 Jira 项目，给我讲讲 JWT 的原理和优缺点。」
- 「用本项目的事务场景，帮我写一份可用于面试的回答。」

**AI 执行步骤：**

1. 先用通俗语言解释概念本身（JWT、事务、乐观锁、Redis 缓存等）。
2. 再结合本项目中的实际或拟实现方案：
   - 指出这些概念会出现在项目的哪些模块中。
   - 描述设计决策和权衡（为什么使用这种技术方案）。
3. 最后输出一份**面试回答模版**：
   - 包含：项目背景 → 场景描述 → 方案设计 → 遇到的问题 → 优缺点与思考。

## Skill D：排查 Bug / 错误（调试器）

**触发方式：**

- 「调用登录接口返回 500，日志如下……帮我排查。」
- 「前端访问项目列表接口返回 403，帮我看下是不是权限写错了。」

**AI 执行步骤：**

1. 仔细阅读你提供的错误日志、请求信息、异常栈等。
2. 使用代码阅读工具定位相关 Controller / Service / 配置文件。
3. 按顺序排查：
   - 配置是否正确（yml、环境变量、依赖版本）。
   - 接口路径和请求方法是否与前端一致。
   - 权限与鉴权逻辑是否符合预期。
   - 业务逻辑和数据是否存在边界情况未处理。
4. 给出修复建议，并在你允许的情况下直接修改代码：
   - 说明修改原因和位置。
   - 提供本地验证步骤（调用示例、预期返回）。

## Skill E：阶段总结与项目包装（总结器）

**触发方式：**

- 「阶段 1 差不多完成了，帮我写一段阶段总结和项目介绍，用于面试。」
- 「帮我把目前的项目整理成一份对外的项目说明。」

**AI 执行步骤：**

1. 根据学习计划文档中相应阶段的目标与任务，结合你已经完成的功能，总结：
   - 本阶段的功能成果。
   - 掌握的技术点与对应模块。
2. 输出两部分内容：
   - 给你自己看的学习总结提纲（方便你写自己的笔记）。
   - 面试可直接使用的项目介绍稿，语言简洁、结构清晰。
3. 如你要求，可进一步根据这些内容生成：
   - 项目 README 草案（如你明确表示需要）。
   - 面试问答清单（针对本项目可能被问到的问题）。

---

> 使用方法小结：你不需要记住这些 SKILL 的名字，只需要用自然语言描述「阶段 + 任务 + 期望行为」，AI 就会自动对照此文件的约定，以统一、稳定的风格来帮你推进迷你 Jira 项目。
